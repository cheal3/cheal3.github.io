1.컴포넌트를 표현하는 JSX

JSX는 JavaScript XML의 줄임말로 자바스크립트에 XML을 추가한 확장형 문법이다.

JSX는 하나의 파일에 자바스크립트와 HTML을 동시에 작성 할 수 있어 편리하다.


src 폴더 아래에 App.js를 열어 아래의 내용을 작성해 보자 . return() 함수 내에 HTML을 사용했다는 점을 알 수 있으며, img 엘리멘트에 /> 와 같은 표현도 보일 것이다. 이것이 바로 JSX이다.


리액트 엔진은 JSX의 XML 구조를 분석하여 자바스크립트 함수 코드로 변환한다.

이는 개발자가 화면 구성에만 집중할 수 있도록 도와준다.


2.컴포넌트와 구성 요소

기존의 웹 프레임워크는 MVC방식으로 정보 , 화면 , 구동 코드를 분리하여 관리했다. 정보를 담당하는 모델 , 화면을 담당하는 뷰 , 구동을 담당하는 컨트롤러에서 MVC라는 용어가 나왔다. 이 방식은 코드를 효율적으로 관리할 수 있다는 장점이 있으나 의존성이 높아 재활용이 어려웠다. (하나만 바꾸기가 쉽지 않았다.)

하지만 웹 사이트의 화면은 각 요소가 비슷하고 반복적으로 사용한 경우가 많다. 이점을 착안하여 등장한 것이 컴포넌트이다. 컴포넌트는 MVC의 뷰를 독립적으로 구성하여 재사용도 할 수 있고 컴포넌트를 통해 새로운 컴포넌트를 만들기도 용이하다.

프로퍼티의 사용 방법

프로퍼티에서는 자바스크립트의 자료형을 모두 사용할 수 있다. 이때 프로퍼티의 자료형은 미리 선언해 주는 것이 좋다. 프로퍼티의 자료형을 미리 선언하면 리액트 엔진이 프로퍼티로 전달하는 값의 변화를 효율적으로 감지할 수 있고, 개발자가 실수로 지정되지 않은 자료형을 전달하면 경고 메시지로 알려주기 때문이다.

State를 사용할 때 주의할 점

  1. 생성자에서 반드시 초기화해야 한다.
  2. state값을 변경할 때는 setState()함수를 반드시 사용해야 한다.
  3. setState()함수는 비동기로 처리되며 , setState()코드 이후로 연결된 함수들의 실행이 완료된 시점에 화면 동기화 과정을 거친다.
  1. state에 저장되는 객체는 반드시 초기화해야 합니다. 그렇지 않으면 내부 함수에서 state값에 접근할 수 없다. 마땅한 초깃값이 없다면 빈 객체라도 넣어야 한다.
  2. state값을 직접 변경하면 안 되는 이유는 state값을 직접 변경해도 render() 함수로 화면을 그려주는 시점은 리액트 엔진이 정하기 때문이다. 그 말은 state를 직접 변경해도 render 함수가 새로 호출되지 않는다. setState()를 통해 state 값을 변경하면 리액트 엔진이 자동으로 render()를 호출하므로 화면에 변경된 값이 잘 적용될 것이다.
  3. setState() 함수의 인자로 함수를 전달하면 이전 state값을 따로 읽는 과정을 생략할 수 있다. 예제를 보자
